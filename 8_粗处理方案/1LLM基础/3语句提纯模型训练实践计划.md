# 语句提纯模型训练实践计划

> 针对"训练一个专门用于语句提纯/意图标准化的轻量 LLM"的学习路径、技术栈选型和相关文献整理

## 一、模型训练学习路径

### 1.1 需要的基础知识（按学习顺序）

#### 第一阶段：Python + 深度学习基础（1-2 周）

你已经有 LangGraph 和 LLM 应用层经验，Python 应该没问题。需要补的是深度学习训练侧的基础：

| 知识点 | 说明 | 推荐资源 |
|--------|------|---------|
| PyTorch 基础 | 张量操作、自动求导、训练循环 | [PyTorch 官方教程](https://pytorch.org/tutorials/) |
| Transformer 架构原理 | Attention、Self-Attention、位置编码 | 李宏毅 2024 机器学习课程（B站免费） |
| Tokenizer | BPE、SentencePiece，理解 token 是什么 | Hugging Face NLP Course Chapter 6 |
| 损失函数与优化器 | CrossEntropyLoss、AdamW、学习率调度 | 跟着 PyTorch 教程走即可 |

> **重点**：你不需要从零训练模型，只需要理解训练过程的原理，核心工作是**微调**（Fine-tuning），不是预训练。

#### 第二阶段：Hugging Face 生态（1 周）

这是你最核心的技术栈，后续所有工作都围绕它展开：

| 工具 | 用途 |
|------|------|
| `transformers` | 模型加载、推理、Tokenizer |
| `datasets` | 数据集加载与处理 |
| `peft` | LoRA / QLoRA 等参数高效微调 |
| `trl` | SFT（Supervised Fine-Tuning）训练器 |
| `accelerate` | 多卡/混合精度训练 |

推荐学习路径：
1. [Hugging Face NLP Course](https://huggingface.co/learn/nlp-course)（免费，官方出品）
2. 重点看 Chapter 7: Fine-tuning（直接对标你的需求）

#### 第三阶段：微调实践（1-2 周）

| 步骤 | 具体内容 |
|------|---------|
| 数据准备 | 用大模型生成训练数据 → 清洗 → 转为 instruction 格式 |
| 选择基座模型 | Qwen2.5-0.5B / Qwen2.5-1.5B（推荐起步） |
| 微调方法 | QLoRA（4bit 量化 + LoRA，显存需求最低） |
| 训练 | 用 `trl` 的 `SFTTrainer` 一键训练 |
| 评估 | 在验证集上测准确率 |
| 部署 | 合并 LoRA 权重 → 导出 → 用 vLLM/Ollama 部署 |

### 1.2 完整技术栈总览

```
数据层
├── 大模型 API（GPT-4o-mini / Qwen-Plus）→ 生成标注数据
├── Python 脚本 → 数据清洗、格式转换
└── Hugging Face datasets → 数据加载

训练层
├── 基座模型：Qwen2.5-0.5B 或 Qwen2.5-1.5B
├── 微调框架：Hugging Face transformers + peft + trl
├── 微调方法：QLoRA（4-bit 量化，显存友好）
└── 训练监控：Weights & Biases（wandb，免费）或 TensorBoard

部署层
├── 本地推理：Ollama（最简单）/ llama.cpp
├── 服务化推理：vLLM（高吞吐）/ TGI
└── 集成到 LangGraph：自定义 LLM 节点调用本地模型
```

## 二、算力方案

### 2.1 你的任务算力需求评估

语句提纯是一个**轻量任务**，模型参数量小（0.5B-1.5B），训练数据量不大（几千到万级），所以算力需求远低于预训练或大模型微调。

| 模型规模 | 微调方法 | 最低显存需求 | 训练时间（1万条数据） |
|----------|---------|------------|-------------------|
| 0.5B | QLoRA 4-bit | **6 GB** | ~30 分钟 |
| 1.5B | QLoRA 4-bit | **8 GB** | ~1 小时 |
| 7B | QLoRA 4-bit | **16 GB** | ~3-4 小时 |

### 2.2 三种算力方案对比

#### 方案一：本地显卡（推荐起步）

如果你有一张游戏显卡：

| 显卡 | 显存 | 能训练的模型 | 二手参考价 |
|------|------|------------|-----------|
| RTX 3060 | 12GB | 0.5B-1.5B 轻松，7B 勉强 | ~1200 元 |
| RTX 3090 | 24GB | 7B 甚至 14B | ~3500 元 |
| RTX 4060 Ti 16G | 16GB | 0.5B-7B | ~3000 元 |

优点：一次投入，长期使用，调试方便
缺点：初始成本

> **你的场景（0.5B-1.5B 模型）**：一张 RTX 3060 12GB 就完全够用。

#### 方案二：云 GPU 租用（推荐预算有限时）

| 平台 | GPU | 价格 | 说明 |
|------|-----|------|------|
| [AutoDL](https://www.autodl.com) | RTX 3090 24GB | ~1.5 元/小时 | 国内最常用，支持镜像 |
| [Featurize](https://featurize.cn) | RTX 3090 | ~2 元/小时 | 国内平台 |
| Google Colab Pro | T4 / A100 | ~70 元/月 | Jupyter 环境，适合学习 |
| [恒源云](https://gpushare.com) | RTX 3090 | ~1.8 元/小时 | 国内平台 |

> **推荐 AutoDL**：注册即用，按小时计费，训练一次 0.5B 模型大概花费 1-2 元。

#### 方案三：免费资源（适合学习阶段）

| 平台 | 免费额度 | 限制 |
|------|---------|------|
| Google Colab 免费版 | T4 16GB | 每天有使用时长限制 |
| Kaggle Notebooks | T4 / P100 | 每周 30 小时 GPU |
| 魔搭社区（ModelScope） | 免费算力 | 需申请 |

> 学习阶段用 Colab / Kaggle 免费额度跑通流程，正式训练再上 AutoDL，是最经济的路径。

### 2.3 推荐路径

```
学习阶段 → Google Colab / Kaggle 免费 GPU
    │          跑通 Hugging Face 微调流程
    ▼
实验阶段 → AutoDL 租用 3090（几块钱跑一次）
    │          调参、对比不同基座模型效果
    ▼
生产阶段 → 本地部署推理（Ollama / CPU 即可）
               训练好的 0.5B 模型推理不需要 GPU
```

## 三、微调实战最小流程（Quick Start）

以下是一个你可以直接上手的最小闭环，帮助你理解整个流程：

### 3.1 用大模型生成训练数据

```python
# 调用大模型 API 批量生成标注数据
prompt = """请将以下用户问句提取为结构化意图JSON，格式：
{"school": "", "major": "", "intent": "", "keywords": []}

用户问句：{query}
只输出JSON，不要其他内容。"""

# 对几千条真实/模拟问句调用 GPT-4o-mini 或 Qwen-Plus
# 收集 (input, output) 对作为训练数据
```

### 3.2 数据格式（Alpaca 格式，SFTTrainer 常用）

```json
[
  {
    "instruction": "请将以下用户问句提取为结构化意图JSON",
    "input": "我想了解清华大学计算机系的保研要求和流程",
    "output": "{\"school\": \"清华大学\", \"major\": \"计算机\", \"intent\": \"保研要求+流程\", \"keywords\": [\"保研\", \"要求\", \"流程\"]}"
  },
  {
    "instruction": "请将以下用户问句提取为结构化意图JSON",
    "input": "北大数学系保研需要GPA多少",
    "output": "{\"school\": \"北京大学\", \"major\": \"数学\", \"intent\": \"GPA要求\", \"keywords\": [\"保研\", \"GPA\"]}"
  }
]
```

### 3.3 QLoRA 微调代码骨架

```python
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model
from trl import SFTTrainer, SFTConfig
from datasets import load_dataset

# 1. 加载基座模型（4-bit 量化）
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype="float16",
)
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-0.5B-Instruct",
    quantization_config=bnb_config,
    device_map="auto",
)
tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2.5-0.5B-Instruct")

# 2. 配置 LoRA
lora_config = LoraConfig(
    r=16,                     # LoRA 秩，越大拟合能力越强
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],  # 只在 attention 层加 LoRA
    lora_dropout=0.05,
    task_type="CAUSAL_LM",
)

# 3. 加载数据
dataset = load_dataset("json", data_files="train_data.json")

# 4. 训练
trainer = SFTTrainer(
    model=model,
    train_dataset=dataset["train"],
    args=SFTConfig(
        output_dir="./intent-purifier-lora",
        num_train_epochs=3,
        per_device_train_batch_size=4,
        learning_rate=2e-4,
        logging_steps=10,
    ),
    peft_config=lora_config,
)
trainer.train()

# 5. 保存
trainer.save_model("./intent-purifier-lora")
```

### 3.4 部署推理

```bash
# 合并 LoRA 权重后，用 Ollama 部署（CPU 即可运行 0.5B 模型）
ollama create intent-purifier -f Modelfile
ollama run intent-purifier "清华计算机保研需要什么条件"
```

## 四、相关文献与学习材料

### 4.1 知识蒸馏（Knowledge Distillation）

| 文献/资源 | 说明 |
|-----------|------|
| **Hinton et al., 2015** — *"Distilling the Knowledge in a Neural Network"* | 知识蒸馏的奠基论文，提出 Teacher-Student 框架。理解核心思想即可，不需要看数学细节 |
| **Alpaca (Stanford, 2023)** — *"Self-Instruct: Aligning Language Models with Self-Generated Instructions"* | 用 GPT 生成指令数据训练小模型的经典案例，和你的"大模型教小模型"思路一致 |
| **Orca (Microsoft, 2023)** — *"Orca: Progressive Learning from Complex Explanation Traces of GPT-4"* | 微软的知识蒸馏实践，强调让小模型学习大模型的推理过程而非仅学结果 |
| **Zephyr (HuggingFace, 2023)** — *"Zephyr: Direct Distillation of LM Alignment"* | Hugging Face 团队的蒸馏实践，用 DPO 训练，参考其数据构造方法 |

> **推荐阅读顺序**：Alpaca → Orca → Zephyr（从简单到复杂）

### 4.2 意图识别与 Slot Filling

| 文献/资源 | 说明 |
|-----------|------|
| **Liu & Lane, 2016** — *"Attention-Based Recurrent Neural Network Models for Joint Intent Detection and Slot Filling"* | 联合意图检测+槽位填充的经典论文，理解 Slot Filling 这个任务的定义 |
| **Chen et al., 2019** — *"BERT for Joint Intent Classification and Slot Filling"* | 用 BERT 做意图识别，架构简单易懂，是 NLU 领域的 baseline |
| **Qin et al., 2021** — *"A Survey on Spoken Language Understanding: Recent Advances and New Frontiers"* | 综述论文，全面了解 Intent Detection + Slot Filling 的发展脉络 |
| **Rasa NLU 文档** — [rasa.com/docs/rasa/nlu](https://rasa.com/docs/rasa/nlu-training-data/) | 工业级开源 NLU 框架，看它如何定义意图和槽位，有实际参考价值 |

### 4.3 Query Understanding / Query Rewriting

| 文献/资源 | 说明 |
|-----------|------|
| **Google, 2020** — *"Query Understanding for Search Engines"* (Croft et al.) | 搜索引擎中 Query 理解的综合介绍 |
| **Yu et al., 2020** — *"Few-Shot Generative Conversational Query Rewriting"* | 对话式查询改写，和你的"语句提纯"目标很接近 |
| **Ma et al., 2023** — *"Query Rewriting in Retrieval-Augmented Large Language Models"* | 直接在 RAG 场景下做 Query Rewriting，和你的系统架构高度相关 |

### 4.4 语义缓存（Semantic Cache）

| 文献/资源 | 说明 |
|-----------|------|
| **GPTCache** — [github.com/zilliztech/GPTCache](https://github.com/zilliztech/GPTCache) | 当前最完整的 LLM 语义缓存开源框架，看源码理解架构 |
| **Bang et al., 2023** — *"GPTCache: An Open-Source Semantic Cache for LLM Applications"* | GPTCache 的技术论文 |
| **Zhu et al., 2023** — *"CacheGen: KV Cache Compression and Streaming for Fast Large Language Model Serving"* | 另一种缓存思路（KV Cache 层面），拓展视野 |

### 4.5 Dialogue State Tracking（对话状态追踪）

| 文献/资源 | 说明 |
|-----------|------|
| **Wu et al., 2019** — *"Transferable Multi-Domain State Generator for Task-Oriented Dialogue Systems"* (TRADE) | DST 领域经典，理解"对话状态"的概念 |
| **MultiWOZ Dataset** — [github.com/budzianowski/multiwoz](https://github.com/budzianowski/multiwoz) | 最常用的 DST 基准数据集，可以用来理解对话状态的标注方式 |
| **Hugging Face Task-Oriented Dialogue** — [相关示例](https://huggingface.co/tasks) | 实战代码参考 |

## 六、关键提醒

1. **先跑通再优化**：第一版不追求效果完美，先用最简单的数据和默认参数跑通全流程
2. **数据质量 > 数据数量**：1000 条高质量标注 > 10000 条噪声数据
3. **0.5B 够用就不要用 7B**：语句提纯是轻量任务，小模型推理快、部署简单
4. **保留大模型兜底**：正如你自己设计的，小模型不确定时回退给大模型，系统容错性更强
