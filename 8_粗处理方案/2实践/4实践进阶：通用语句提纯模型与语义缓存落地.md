## 二、第一步：定义通用语句提纯规范（最重要）

这是整个方案的根基。你现在的保研版 schema 是：

```json
{"school": "清华大学", "major": "计算机", "intent": "保研要求", "keywords": ["保研", "要求"]}
```

这只能用于保研场景。要做通用的语句提纯，需要设计一个**不依赖业务领域的抽象 schema**。

### 2.1 通用提纯 Schema 设计

```json
{
  "intent_type": "query",
  "action": "查询条件",
  "entities": [
    {"type": "organization", "value": "清华大学"},
    {"type": "field", "value": "计算机"},
    {"type": "topic", "value": "保研"}
  ],
  "constraints": ["GPA要求", "流程"],
  "normalized_query": "清华大学 计算机专业 保研条件和流程"
}
```

各字段说明：

| 字段 | 作用 | 取值范围 | 示例 |
|------|------|---------|------|
| `intent_type` | 语句的目的分类 | `query`（查询）/ `compare`（对比）/ `recommend`（推荐）/ `action`（操作）/ `chat`（闲聊） | "保研需要什么" → `query` |
| `action` | 具体要执行的动作 | 自由文本，动宾短语 | "查询条件"、"对比学校"、"推荐方案" |
| `entities` | 提取的关键实体 | 数组，每个包含 `type` + `value` | 学校、专业、人名、产品名等 |
| `constraints` | 附加约束/限定 | 字符串数组 | "GPA要求"、"2024年"、"理科生" |
| `normalized_query` | 标准化后的简洁表述 | 一句话 | 去除口语化、重复、语气词后的核心表述 |

### 2.2 entity type 的通用设计

不要为每个业务定义专属的 entity type，而是用**通用的 NER 类别**：

| entity type | 说明 | 示例 |
|-------------|------|------|
| `organization` | 组织/机构/学校/公司 | 清华大学、腾讯、教育部 |
| `field` | 领域/专业/学科/行业 | 计算机、经济学、人工智能 |
| `person` | 人名 | 张三、李教授 |
| `location` | 地点 | 北京、海淀区 |
| `time` | 时间 | 2024年、下学期、近三年 |
| `topic` | 主题/事件 | 保研、考研、实习、比赛 |
| `metric` | 指标/数值 | GPA 3.5、排名前10% |
| `product` | 产品/服务 | iPhone、ChatGPT |
| `other` | 无法归类的实体 | 兜底 |

这套 entity type 在保研场景、电商场景、客服场景中都通用。

### 2.3 不同场景下的 schema 输出示例

**保研场景**：
```
输入："清华计算机保研需要GPA多少"
输出：{
  "intent_type": "query",
  "action": "查询GPA要求",
  "entities": [
    {"type": "organization", "value": "清华大学"},
    {"type": "field", "value": "计算机"},
    {"type": "topic", "value": "保研"}
  ],
  "constraints": ["GPA"],
  "normalized_query": "清华大学 计算机 保研 GPA要求"
}
```

**电商场景**：
```
输入："iPhone 16 Pro Max 256G 现在多少钱，有没有优惠"
输出：{
  "intent_type": "query",
  "action": "查询价格和优惠",
  "entities": [
    {"type": "product", "value": "iPhone 16 Pro Max"},
    {"type": "metric", "value": "256G"}
  ],
  "constraints": ["当前价格", "优惠活动"],
  "normalized_query": "iPhone 16 Pro Max 256G 价格及优惠"
}
```

**客服场景**：
```
输入："我昨天买的东西怎么还没发货啊，订单号是 20240520001"
输出：{
  "intent_type": "query",
  "action": "查询物流状态",
  "entities": [
    {"type": "time", "value": "昨天"},
    {"type": "other", "value": "订单号 20240520001"}
  ],
  "constraints": ["发货状态"],
  "normalized_query": "订单20240520001 物流发货状态查询"
}
```

**闲聊/无意图**：
```
输入："哈哈哈你好搞笑"
输出：{
  "intent_type": "chat",
  "action": "闲聊",
  "entities": [],
  "constraints": [],
  "normalized_query": "闲聊"
}
```

### 2.4 Schema 设计的关键原则

1. **宁简勿细**：entity type 控制在 10 种以内，太多小模型学不好
2. **兜底字段**：`other` type 和 `chat` intent_type 保证任何输入都有合法输出
3. **normalized_query 是核心**：这个字段用于后续的缓存匹配，是提纯的最终产物
4. **JSON 输出必须可校验**：后续流程依赖 JSON 解析，格式错误直接回退大模型

### 3.2 用大模型批量生成训练数据的脚本

```python
import json
import random
from openai import OpenAI  # 或用 dashscope 调 Qwen-Plus

client = OpenAI(api_key="your-key", base_url="your-endpoint")

DOMAINS = [
    ("高校保研", ["清华大学计算机保研要求", "北大物理考研难度", "复旦推免名额", "浙大直博条件"]),
    ("电商购物", ["iPhone价格", "笔记本电脑推荐", "退货流程", "优惠券怎么用"]),
    ("客服咨询", ["订单没发货", "退款要多久", "怎么联系人工客服", "投诉商家"]),
    ("技术问答", ["Python怎么读文件", "Vue和React区别", "Docker部署教程", "Git冲突解决"]),
    ("日常闲聊", ["今天天气怎么样", "你好", "哈哈有意思", "推荐个电影"]),
    ("医疗健康", ["头疼怎么办", "感冒要吃什么药", "体检项目有哪些", "挂号怎么挂"]),
    ("旅游出行", ["北京三日游攻略", "签证怎么办", "酒店推荐", "机票什么时候最便宜"]),
    ("金融理财", ["基金怎么买", "房贷利率多少", "信用卡额度怎么提", "股票开户流程"]),
]

GENERATION_PROMPT = """你是一个语句提纯标注助手。请根据给定的领域，生成 5 组训练数据。

每组数据包含：
1. 一个用户可能说的自然语言问句（包含口语化、省略、模糊等真实表述）
2. 对应的结构化意图 JSON

JSON 格式规范：
{
  "intent_type": "query|compare|recommend|action|chat",
  "action": "动宾短语描述具体动作",
  "entities": [{"type": "organization|field|person|location|time|topic|metric|product|other", "value": "实体值"}],
  "constraints": ["附加约束条件"],
  "normalized_query": "标准化后的简洁表述"
}

领域：{domain}
参考问句方向：{examples}

要求：
- 问句要多样化，包含不同表达方式（正式/口语/省略/带语气词）
- entities 只提取真正存在的实体，没有就留空数组
- normalized_query 要简洁准确，去除语气词和冗余

请输出 JSON 数组格式：
[{{"input": "用户问句", "output": {{结构化JSON}}}}, ...]"""

all_data = []

for domain, examples in DOMAINS:
    for batch in range(10):  # 每个领域生成 10 批，共 50 条
        prompt = GENERATION_PROMPT.format(
            domain=domain,
            examples=", ".join(random.sample(examples, min(3, len(examples))))
        )
        response = client.chat.completions.create(
            model="gpt-4o-mini",  # 或 qwen-plus
            messages=[{"role": "user", "content": prompt}],
            temperature=0.9,
        )
        try:
            batch_data = json.loads(response.choices[0].message.content)
            all_data.extend(batch_data)
            print(f"[{domain}] batch {batch+1}: 生成 {len(batch_data)} 条")
        except json.JSONDecodeError:
            print(f"[{domain}] batch {batch+1}: JSON 解析失败，跳过")

print(f"\n总计生成 {len(all_data)} 条训练数据")

# 保存
with open("universal_intent_train_raw.json", "w", encoding="utf-8") as f:
    json.dump(all_data, f, ensure_ascii=False, indent=2)
```

### 3.3 数据清洗与校验

生成的数据不能直接用，必须清洗：

```python
import json

def validate_sample(sample):
    """校验一条训练数据是否合法"""
    errors = []

    if "input" not in sample or not sample["input"].strip():
        errors.append("缺少 input 或 input 为空")
        return errors

    output = sample.get("output", {})
    if isinstance(output, str):
        try:
            output = json.loads(output)
        except:
            errors.append("output 不是合法 JSON")
            return errors

    valid_intents = {"query", "compare", "recommend", "action", "chat"}
    if output.get("intent_type") not in valid_intents:
        errors.append(f"intent_type 无效: {output.get('intent_type')}")

    if not output.get("action"):
        errors.append("缺少 action")

    if not isinstance(output.get("entities", []), list):
        errors.append("entities 不是数组")

    valid_entity_types = {"organization", "field", "person", "location", "time", "topic", "metric", "product", "other"}
    for ent in output.get("entities", []):
        if ent.get("type") not in valid_entity_types:
            errors.append(f"entity type 无效: {ent.get('type')}")
        if not ent.get("value"):
            errors.append("entity 缺少 value")

    if not output.get("normalized_query"):
        errors.append("缺少 normalized_query")

    return errors


with open("universal_intent_train_raw.json", "r", encoding="utf-8") as f:
    raw_data = json.load(f)

clean_data = []
error_count = 0

for sample in raw_data:
    errors = validate_sample(sample)
    if errors:
        error_count += 1
    else:
        clean_data.append(sample)

print(f"原始数据: {len(raw_data)} 条")
print(f"校验通过: {len(clean_data)} 条")
print(f"校验失败: {error_count} 条")

# 转为 SFTTrainer 格式
sft_data = []
for sample in clean_data:
    output = sample["output"] if isinstance(sample["output"], str) else json.dumps(sample["output"], ensure_ascii=False)
    sft_data.append({
        "instruction": "请将以下用户问句提取为结构化意图JSON，格式包含 intent_type, action, entities, constraints, normalized_query",
        "input": sample["input"],
        "output": output,
    })

# 划分训练集和验证集（90% / 10%）
import random
random.shuffle(sft_data)
split = int(len(sft_data) * 0.9)
train_data = sft_data[:split]
val_data = sft_data[split:]

with open("train.json", "w", encoding="utf-8") as f:
    json.dump(train_data, f, ensure_ascii=False, indent=2)

with open("val.json", "w", encoding="utf-8") as f:
    json.dump(val_data, f, ensure_ascii=False, indent=2)

print(f"训练集: {len(train_data)} 条")
print(f"验证集: {len(val_data)} 条")
```

### 3.4 数据量建议

| 阶段 | 数据量 | 效果预期 |
|------|--------|---------|
| MVP 验证 | 500-1000 条 | 能跑通，常见场景基本对 |
| 基本可用 | 2000-5000 条 | 主流场景准确率 80%+ |
| 生产级别 | 5000-10000 条 | 各领域覆盖充分，边缘 case 减少 |

建议先生成 2000-3000 条开始训练，根据验证集的结果决定是否需要补充数据。

## 四、第三步：重新训练通用版模型

### 4.1 训练代码（基于你已有经验，调整如下）

和你之前训练保研版的代码基本一样，主要变化是数据和超参数：

```python
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig
from peft import LoraConfig
from trl import SFTTrainer, SFTConfig
from datasets import load_dataset

model_name = "Qwen/Qwen2.5-0.5B-Instruct"

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype="float16",
)

model = AutoModelForCausalLM.from_pretrained(
    model_name, quantization_config=bnb_config, device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained(model_name)

# 相比保研版，扩大了 LoRA 的作用范围
lora_config = LoraConfig(
    r=32,                # 从 16 提升到 32，因为通用任务更复杂
    lora_alpha=64,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],  # 增加了 k_proj 和 o_proj
    lora_dropout=0.05,
    task_type="CAUSAL_LM",
)

dataset = load_dataset("json", data_files={"train": "train.json", "validation": "val.json"})

trainer = SFTTrainer(
    model=model,
    train_dataset=dataset["train"],
    eval_dataset=dataset["validation"],
    args=SFTConfig(
        output_dir="./universal-intent-purifier-lora",
        num_train_epochs=5,           # 通用任务适当多训几轮
        per_device_train_batch_size=4,
        learning_rate=1e-4,           # 比保研版略低，避免过拟合
        logging_steps=10,
        eval_strategy="steps",
        eval_steps=50,                # 每 50 步在验证集上评估一次
        save_strategy="steps",
        save_steps=100,
        load_best_model_at_end=True,  # 训练结束自动加载最佳 checkpoint
        metric_for_best_model="eval_loss",
    ),
    peft_config=lora_config,
)

trainer.train()
trainer.save_model("./universal-intent-purifier-lora")
```

### 4.2 评估方法

训练完后，用验证集评估模型质量：

```python
import json
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import PeftModel

model_name = "Qwen/Qwen2.5-0.5B-Instruct"
tokenizer = AutoTokenizer.from_pretrained(model_name)
base_model = AutoModelForCausalLM.from_pretrained(model_name, torch_dtype=torch.float16, device_map="auto")
model = PeftModel.from_pretrained(base_model, "./universal-intent-purifier-lora")

with open("val.json", "r", encoding="utf-8") as f:
    val_data = json.load(f)

total = 0
json_valid = 0
intent_correct = 0
entity_correct = 0

for sample in val_data:
    messages = [
        {"role": "system", "content": "请将以下用户问句提取为结构化意图JSON，格式包含 intent_type, action, entities, constraints, normalized_query"},
        {"role": "user", "content": sample["input"]},
    ]
    text = tokenizer.apply_chat_template(messages, tokenize=False, add_generation_prompt=True)
    inputs = tokenizer(text, return_tensors="pt").to(model.device)

    with torch.no_grad():
        outputs = model.generate(**inputs, max_new_tokens=300, temperature=0.1, do_sample=False)

    response = tokenizer.decode(outputs[0][inputs["input_ids"].shape[1]:], skip_special_tokens=True)

    total += 1
    expected = json.loads(sample["output"]) if isinstance(sample["output"], str) else sample["output"]

    try:
        predicted = json.loads(response)
        json_valid += 1

        if predicted.get("intent_type") == expected.get("intent_type"):
            intent_correct += 1

        pred_entities = {(e["type"], e["value"]) for e in predicted.get("entities", [])}
        exp_entities = {(e["type"], e["value"]) for e in expected.get("entities", [])}
        if pred_entities == exp_entities:
            entity_correct += 1

    except json.JSONDecodeError:
        pass

print(f"总样本数: {total}")
print(f"JSON 格式正确率: {json_valid/total*100:.1f}%")
print(f"intent_type 准确率: {intent_correct/total*100:.1f}%")
print(f"entities 完全匹配率: {entity_correct/total*100:.1f}%")
```

**目标指标**：

| 指标 | 最低要求 | 良好 |
|------|---------|------|
| JSON 格式正确率 | > 95% | > 99% |
| intent_type 准确率 | > 85% | > 92% |
| entities 完全匹配率 | > 75% | > 85% |

达不到最低要求 → 增加训练数据 / 调整超参数 / 检查数据质量

## 六、第五步：搭建语义缓存系统

### 6.1 缓存数据库选型

| 方案 | 适用场景 | 说明 |
|------|---------|------|
| **SQLite + JSON** | 开发/小规模 | 零依赖，单文件数据库，先用这个跑通 |
| **Redis + RedisJSON** | 中等规模 | 内存缓存，快速读写，适合生产 |
| **PostgreSQL + pgvector** | 需要向量检索时 | 同时支持结构化查询和向量相似度 |

建议：**先用 SQLite 跑通流程，后续按需切换 Redis。**

### 6.2 缓存数据结构设计

```sql
CREATE TABLE intent_cache (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    
    -- 提纯后的结构化意图（用于硬匹配）
    intent_type TEXT NOT NULL,
    action TEXT NOT NULL,
    entities_json TEXT NOT NULL,        -- JSON 字符串
    constraints_json TEXT NOT NULL,     -- JSON 字符串
    normalized_query TEXT NOT NULL,     -- 用于软匹配 / 展示
    
    -- 原始问答对
    original_query TEXT NOT NULL,       -- 用户原始问句
    answer TEXT NOT NULL,               -- LLM 生成的答案
    
    -- 缓存管理
    hit_count INTEGER DEFAULT 0,        -- LFU 命中次数
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_hit_at TIMESTAMP,
    
    -- 索引
    INDEX idx_intent (intent_type, action),
    INDEX idx_entities (entities_json),
    INDEX idx_hit_count (hit_count)
);
```

### 6.3 缓存匹配逻辑（硬匹配 + 软匹配）

```python
import json
import sqlite3
from difflib import SequenceMatcher

def find_cache_match(purified_intent: dict, db_path: str = "cache.db", similarity_threshold: float = 0.80):
    """
    两层匹配：
    1. 硬匹配：intent_type + entities 精确一致
    2. 软匹配：normalized_query 文本相似度
    """
    conn = sqlite3.connect(db_path)
    cursor = conn.cursor()

    # 第一层：硬匹配 intent_type
    cursor.execute(
        "SELECT * FROM intent_cache WHERE intent_type = ?",
        (purified_intent["intent_type"],)
    )
    candidates = cursor.fetchall()

    if not candidates:
        conn.close()
        return None, 0.0

    best_match = None
    best_score = 0.0

    for row in candidates:
        cache_entities = json.loads(row[3])  # entities_json
        input_entities = purified_intent.get("entities", [])

        # 硬匹配：entities 中的关键实体必须一致
        cache_entity_set = {(e["type"], e["value"]) for e in cache_entities}
        input_entity_set = {(e["type"], e["value"]) for e in input_entities}

        # 计算 entity 重叠度
        if cache_entity_set and input_entity_set:
            entity_overlap = len(cache_entity_set & input_entity_set) / max(len(cache_entity_set), len(input_entity_set))
        elif not cache_entity_set and not input_entity_set:
            entity_overlap = 1.0
        else:
            entity_overlap = 0.0

        if entity_overlap < 0.8:
            continue  # entity 不匹配，跳过

        # 第二层：软匹配 normalized_query
        cache_query = row[5]  # normalized_query
        input_query = purified_intent.get("normalized_query", "")
        text_similarity = SequenceMatcher(None, cache_query, input_query).ratio()

        # 综合得分 = entity重叠度 * 0.6 + 文本相似度 * 0.4
        score = entity_overlap * 0.6 + text_similarity * 0.4

        if score > best_score:
            best_score = score
            best_match = row

    conn.close()

    if best_score >= similarity_threshold:
        return {
            "answer": best_match[7],             # answer
            "original_query": best_match[6],     # original_query
            "hit_count": best_match[8],          # hit_count
            "coverage": round(best_score * 100, 1),  # 覆盖率百分比
        }, best_score
    else:
        return None, best_score
```

### 6.4 LFU 淘汰策略

```python
def evict_lfu(db_path: str = "cache.db", max_cache_size: int = 100):
    """当缓存超过 max_cache_size 时，淘汰命中次数最少的记录"""
    conn = sqlite3.connect(db_path)
    cursor = conn.cursor()

    cursor.execute("SELECT COUNT(*) FROM intent_cache")
    count = cursor.fetchone()[0]

    if count > max_cache_size:
        delete_count = count - max_cache_size
        cursor.execute("""
            DELETE FROM intent_cache WHERE id IN (
                SELECT id FROM intent_cache
                ORDER BY hit_count ASC, last_hit_at ASC
                LIMIT ?
            )
        """, (delete_count,))
        conn.commit()
        print(f"LFU 淘汰: 删除 {delete_count} 条低频缓存")

    conn.close()
```

## 七、第六步：LangGraph 编排

### 7.1 整体 Graph 设计

```
用 LangGraph 定义的 StateGraph：

节点（Nodes）：
├── purify_intent      → 调用本地提纯模型（Ollama）
├── validate_purified  → 校验 JSON 输出是否合法
├── fallback_purify    → 大模型兜底提纯
├── search_cache       → 缓存匹配
├── generate_answer    → 大模型生成答案（带 RAG）
├── save_to_cache      → 写入缓存
└── return_response    → 返回结果给用户

边（Edges）：
purify_intent → validate_purified
validate_purified → [合法] → search_cache
validate_purified → [不合法] → fallback_purify → search_cache
search_cache → [命中] → return_response（附覆盖率）
search_cache → [未命中] → generate_answer → save_to_cache → return_response
```

### 7.2 TypeScript 伪代码（LangGraph.js）

```typescript
import { StateGraph, START, END } from "@langchain/langgraph";

interface PurifierState {
  userQuery: string;
  purifiedIntent: Record<string, any> | null;
  cacheResult: { answer: string; coverage: number } | null;
  finalAnswer: string;
  usedCache: boolean;
}

const graph = new StateGraph<PurifierState>({
  channels: {
    userQuery: { value: (a, b) => b ?? a },
    purifiedIntent: { value: (a, b) => b ?? a, default: () => null },
    cacheResult: { value: (a, b) => b ?? a, default: () => null },
    finalAnswer: { value: (a, b) => b ?? a, default: () => "" },

})
  .addNode("purifyIntent", async (state) => {

    });

    try {
      return { purifiedIntent: JSON.parse(result.response) };
    } catch {
      return { purifiedIntent: null };
    }
  })
  .addNode("fallbackPurify", async (state) => {
    // 调用大模型做兜底提纯
    // ... 调用 GPT-4o-mini 或 Qwen-Plus
  })
  .addNode("searchCache", async (state) => {
    // 调用缓存匹配 API
    // ... 返回 cacheResult 或 null
  })
  .addNode("generateAnswer", async (state) => {
    // RAG + 大模型生成
    // ... 基于 purifiedIntent 的 keywords 做 RAG
  })
  .addNode("saveToCache", async (state) => {
    // 把新答案写入缓存
  })

  // 边的连接
  .addEdge(START, "purifyIntent")
  .addConditionalEdges("purifyIntent", (state) =>
    state.purifiedIntent ? "searchCache" : "fallbackPurify"
  )
  .addEdge("fallbackPurify", "searchCache")

  )
  .addEdge("generateAnswer", "saveToCache")
  .addEdge("saveToCache", END);
```
